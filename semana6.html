jQuery part II

El Modelo de Objetos de Documento

Para sacarle el máximo partido a jQuery, debemos repasar como se compone una página de HTML.

Un documento de HTML está estructurado de acuerdo con el Document Object Model, es decir, el Modelo de Objetos de Documento, o DOM. jQuery puede acceder y modificar el HTML interactuando con el DOM.

El DOM está compuesto de todos los elementos en la página, dispuestos en una jerarquía que refleja la forma en que el documento de HTML está ordenado. ¿Recuerdas cómo imaginábamos que el documento de HTML era como un árbol? Puedes imaginarte que el DOM es algo parecido. Al igual que en un documento de HTML, los elementos en el DOM pueden tener elementos padres, hijos y hermanos.

Tal como div en CSS se refiere al elemento <div> de HTML, el 'div' en jQuery se refiere al mismo elemento <div> en HTML. Imagina que el nombre del elemento se muestra en jQuery igual a como estaba en CSS, solo que ahora aparece en medio de comillas simples. Así que, por ejemplo, puedes enfocarte en cualquier elemento de la clase button usando:

$('.button').algunaAcción

    Como recordarás, .button es la forma que usas en CSS para enfocarte en cualquier elemento que en tu archivo de HTML tenga class="button".

¿Qué es jQuery?

jQuery es una biblioteca, o un conjunto de complementos útiles (adds-ons), para el lenguaje de programación JavaScript. Tal vez parezca contradictorio aprender cómo usar una biblioteca antes de aprender el lenguaje en sí, pero existen unas buenas razones para ello.

  • Toma cierto tiempo llegar a sentirse cómodo con JavaScript, y es más fácil manipular los elementos del DOM directamente con jQuery de lo que es con JavaScript. Vamos a comenzar mostrándote jQuery, para ayudarte a crear sitios web geniales mucho más rápido.
  • jQuery brinda una interfaz sencilla para el código JavaScript subyacente. Es más fácil para la mayoría de usuarios aprender primero a manejar jQuery, y después sí aventurarse en los detalles de JavaScript.
  • jQuery es mucho mejor en cuanto a resultados inmediatos y visuales que el JavaScript normal.

Sintaxis de jQuery

Si vienes de ver los paréntesis angulares ordenados de HTML, tal vez te parezca que los paréntesis, las llaves y los punto y coma de jQuery son confusos. Afortunadamente el formato de jQuery es predecible, y siempre se verá así:

$(document).ready(function() {
    $(elementoParaClic).evento(function() {
        $(elementoAfectado).efecto();
    });
});

Nota: Posiciona el mouse sobre el texto de explicación para poder ver la información!!!!!

Explicación.....

En la primera línea tienes listo el documento, en la segunda línea estableces el elemento de HTML sobre el que quieres que el usuario haga clic, y en la tercera línea haces que suceda el efecto que quieras (tal como hacer que se desvanezca un 'div').

Tal vez esto parezca demasiada información, así que vamos a verla detalladamente. Vamos a comenzar con $().
$() es una función (un tipo de acción) que convierte lo que esté en medio de los paréntesis en un objeto de jQuery; es decir, algo con lo que jQuery pueda trabajar. (Más adelante hablaremos sobre las funciones.)

Enlazando los archivos de HTML y JavaScript

Ahora necesitamos asociar nuestra página de HTML con nuestro script de jQuery, de modo que nuestra magia de jQuery afecte nuestro HTML.
Al igual que necesitamos una etiqueta <link> para conectar nuestros HTML y CSS, necesitamos una etiqueta <script> para conectar nuestro HTML y jQuery. La etiqueta se ve así:
<script type="text/javascript" src="src"></script>
Observa que la etiqueta <script> no se autocierra; necesita de una etiqueta </script> de cierre.

Primeros pasos

La sintaxis $(document).ready(); que ya has visto. Funciona de esta manera:

  • $() dice: "¡Oigan, están a punto de pasar cosas que tienen que ver con jQuery!"
  • La parte de document en medio de los paréntesis nos indica que estamos a punto de trabajar en el mismo documento de HTML.
  • .ready(); es una función, o una acción básica, en jQuery. Dice: "¡Voy a hacer algo tan pronto como el documento de HTML esté listo!"
  • La parte en medio de los paréntesis de .ready() es el evento de jQuery, que ocurre tan pronto como el documento de HTML está listo.

De este modo,
    $(document).ready(algo);
Dice: "¡cuando el documento de HTML esté listo, haz algo!"
Observa que .ready(); termina con un punto y coma. Esto le indica a jQuery que terminaste de darle un comando.

El enfoque funcional

Ahora necesitamos agregar algo adentro de nuestra función ready().
Recuerda: cuando hablamos de "función", piensa en "acción." Las funciones son la unidad básica con que se trabaja en jQuery.Es por eso que jQuery incluye la palabra clave function. La sintaxis se ve así:
function(){
        acciones de jQuery;
}

Si agregamos nuestra función dentro de nuestro .ready(), jQuery ejecutará el código de nuestra función tan pronto como se cargue el documento de HTML. La sintaxis se vería de esta manera:
$(document).ready(function() {
        acciones de jQuery;
});

Recuerda que cerramos nuestras sentencias de jQuery con un punto y coma.

Deslizamiento electr(on)ico

Ahora solamente tenemos que incluir una acción en el cuerpo de nuestra función. En este caso, haremos un encabezado divdeslizable y que se haga visible para nuestra página HTML.

En medio de los {} de nuestra function(), vamos a convertir nuestro div en un objeto de jQuery, de modo que jQuery pueda manipularlo. De forma muy parecida a como usamos .ready() en $(document), usaremos la acción .slideDown() en nuestro objeto div de jQuery.

Adentro de los paréntesis de .slideDown, vamos a indicarle qué tan rápido debe deslizarse. Vamos a escribir 'slow'.

Dale click al siguiente tema "Adentro....." y observa que sucede!

Adentro...

Ahora vamos a incluir nuestra palabra clave function junto con dos nuevas acciones, mouseenter() y fadeTo().
mouseenter() produce un cambio cuando mueves el puntero dentro de un elemento de HTML determinado. Por ejemplo:

$(document).ready(function() {
    $('div').mouseenter(function() {
        $('div').hide();
    });
});

Debería hacer que todos los <div> de la página se oculten en cuanto muevas el puntero sobre uno de ellos. (En la próxima lección veremos cómo afectar a solamente un <div> entre muchos.) Por ahora solamente tenemos un <div>, así que este código está bien.
Sin embargo, vamos a ubicar a fadeTo() al interior de mouseenter(), en lugar de hide(). fadeTo() toma dos argumentos, o entradas, en medio de sus paréntesis, separados por una coma: la velocidad a la cual se desvanece el elemento, y la opacidad (o transparencia) que tendrá al desvanecerse. Por ejemplo:

    fadeTo('fast', 0.25);

Hará que el elemento se desvanezca rápidamente hasta un 25% de su opacidad original, volviéndolo muy claro.

Funciones, Parte II: funciones explicadas

Una función está compuesta de tres partes: la palabra clave function, los argumentos que la función tome (que van en medio de () y se separan por comas, si se trata de más de uno), y cualquier acción que la función deba ejecutar (que van en medio de {}). La forma general es:

function(argumento1, argumento2, etc.) {
    Realiza una acción
    Realiza otra acción
    ¡Realiza otra acción más!
}

Una de las cosas buenas de jQuery es que el argumento de una función puede ser casi cualquier cosa, ¡incluso puede ser otra función! Es por eso que .ready() puede incluir a function en medio de sus paréntesis; está tomando como argumento a una función.

Asignar valores

El signo = es usado para asignar valores. Por ejemplo, en jQuery puedes escribir
var $p = $('p');
Con lo cual se almacena el objeto de jQuery $('p') en la variable $p . Una variable es solamente un nombre útil que usamos para almacenar valores de jQuery.
¿Cuál es la razón para hacerlo? Pues bien, a veces nuestro código de jQuery se hace muy largo, y puede llegar a ser buena idea almacenarlo en pequeñas variables para usarlo después. Por ejemplo, si queremos modificar un elemento de HTML que está anidado, podríamos hacer esto:

$(document).ready(function() {
    var $item = $('div ul li ol li');
    $item.fadeOut('fast');
});

¡'this' es importante!

En la última lección, teníamos un código que se veía así:

$(document).ready(function() {
    $('div').mouseenter(function() {
        $('div').hide();
    });
});

La segunda línea está bien: nos dice que cuando pasamos el puntero sobre un div, una acción determinada debería ejecutarse. Sin embargo, $('div').hide(); no solamente ocultará el div sobre el que pasas el puntero; también ocultará todos los div de la página. ¿Cómo podemos indicarle a jQuery que solamente queremos afectar este div en particular?
¡Pues para ello usamos this!
La palabra clave this se refiere al objeto de jQuery con el que estamos trabajando en ese momento. Las reglas de operación son un poco complejas, pero lo importante es entender que si usas un controlador de eventos (ese es el nombre elegante de las acciones como .click() y .mouseenter(), ya que controlan eventos de jQuery) en un elemento, puedes llamar al evento que ocurre (tal como fadeOut()) con $(this), y el evento solamente afectará al elemento con el que haces algo en ese momento (por ejemplo, sobre el que se hace clic o se pasa el puntero).
